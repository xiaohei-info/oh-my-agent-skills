# Code of Conduct

## Our standard

This project is for people sharing reusable agent methods, skills, and packaging practices.

We want the repository to be:
- practical
- rigorous
- respectful
- low-drama
- useful to real operators and builders

## Expected behavior

Contributors and participants are expected to:
- criticize ideas clearly without attacking people
- prefer concrete evidence over vague posturing
- keep discussions relevant to reusable workflows and public packaging
- acknowledge uncertainty honestly
- avoid dumping private or machine-specific assumptions into public artifacts
- help improve clarity, portability, and verification quality

## Unacceptable behavior

Unacceptable behavior includes:
- harassment or personal attacks
- deliberate hostility meant to derail collaboration
- spamming or low-effort promotional content
- publishing private credentials, secrets, or sensitive internal material
- knowingly misrepresenting what a skill or method actually does

## Enforcement

The repository maintainer may remove, edit, or reject contributions that violate this code of conduct or degrade the repo's practical value.

## Scope

This code applies to:
- issues
- pull requests
- discussions
- review comments
- other repository communication surfaces

## Guiding principle

Strong disagreement is acceptable.
Low-signal aggression is not.
The goal is to make the public package better.
